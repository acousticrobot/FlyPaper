Class: color

fly. color

Color maintains a palette of colors defined by the Color Palette. It is reinitialized and repopulated every time the Color Palette is reset. When a color is defined in a palette (ex. 'blue'), its colors are accessible as an array via fly.color.

new color()

The Color Palette initializes fly.color during fly.init After initialization by the color Palette, color is granted info.

<static> palette(args)

Returns the name of the current color palette if no args are passed in. Arguments passed on fly.init will be passed to the color palette. Args can be a string matched against predefined sets of colors in the Color Sets. Args can also be an object containing a name and a color set. The set will be added to the colorSets array, or if the name exists already, the new set will replace the old.

args String, Object

Example
// Set the predefined 'neon' color set on init
fly.init({palette:"neon"})

// Change to the predefined "pastel" color set
fly.color.palette("pastel")

// Change to a custom color set, and redefine fly.colors
// To use the custom colors
fly.color.palette({
     name: "my color set", set: [
         ['ruby','#F04510','#FF7070','#FFD3C0'],
         ['orange','#F28614','#FFB444','#FFE8C0'],
         ['sunshine','#CDB211','#FFFF70','#FFFFC0'],
         ['grass','#42622D','#89C234','#C0FFC0'],
         ['berry','#00597C','#00A9EB','#B0E5FF'],
         ['very berry','#6F006F','#9F3DBF','#FFC0FF'],
         ['clouds','#383633','#A7A097','#FFFFFF']
     ]
})

fly.color.palette()
// returns "my color set"

<static> reserved(word) → {Boolean}

In the event that you name a color in your color palette as something that is already being used, the name will be appended with "_color", for example info will become info_color. To check if a word is reserved, call fly.color.reserved('color name');
